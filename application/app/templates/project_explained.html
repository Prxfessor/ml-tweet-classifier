Explaining the project


Project Briefly Explained

The overall goal of the project was to classify a tweet in the following categories: if it is offensive or not, and if it is offensive, if it is a targeted insult or an un-targeted insult. Finally, if it is a targeted insult, is it targeted to a group, individual or other. To do this we would need to use machine learning algorithms. We used supervised machine learning and built numerous models and employed the use of the one with the highest accuracy. The dataset for training and testing was publicly available online at joeykay9's GitHub page. Using this data that is already labelled we can build models. Then using this model, we can classify a new tweet.

Step 3: Clean up the tweets

In this step we cleaned the data up. We used the "nltk" python library and regular expressions. This consisted of the following sub-steps:

  1. Pre-process

    2. We did a number of sub-steps within pre-processing to ensure that the tweets are understandable by the computer:

    • Get rid of "@user"

      • This is because the "@user" is a generic term used to describe another user on twitter. I assume that this approach was taken to ensure anonymity of the author and subject of the tweet

    • Get rid of "URL"

      • This is because the "URL" has already been subbed in for an actual URL. Therefore we can just remove this as it does not add any information to the classifier

    • Get rid of numbers

      • Numbers aren't required by the classifier to make a prediction. Also, numbers can be arbitrary. It can be from a timestamp or from a quantity. Therefore we can remove this

    • Get rid of punctuation

      • Punctuation is not required as it adds no additional information when the computer tries to understand the text. Thus, this can be removed

    • Get rid of emojis

      • Emojis add no context to the tweet and therefore can also be removed.

    • Get rid of any whitespace/tabs

      • Tabs and whitespace do not add any context or any relevance to a tweet being offensive or not. It can subsequently be removed.

    • Make everything lowercase

      • By making everything lowercase we can normalise the string and process can be smoother. For example the word "FUCK" is equal to "fuck". However this may affect nouns or organisations that are capital, for example "PETA" stands for "People for the Ethical Treatment of Animals" which is an organisation. If we make it lower case, "peta" is not equal to "PETA". Here there is the trade-off we have to consider

  2. Remove stopwords

    3. Stopwords are words that are common in the english language but are unimportant. For example, "the", "is", "a", "an", and etc. These words can be removed. The following stopwords are considered and are removed:

    {{stop_words}}

  3. Get POS

    4. POS (Part of Speech) is where the computer adds a tag to the word depending on the context and the particular word. For example, the word "fishing" is a verb therefore we tag it as such.

  4. Lemmatise

    5. Lemmatising a word allows for the computer to convert it into its base form. For example, "running", "ran" and "run" are all words but have the same meaning. Therefore we can get to its base form "run". This can reduce the processing time as well optimizing the words that appear. There is the option to use a stemmer, but in this case it was more appropriate to use a lemmatiser.

Step 4: For each task vectorise the data and build, run, evaluate, and save the models

In this step vectorised the training data, split it into training and testing, then build the model and run it on the data.

  1. Vectorise the Data

    2. In this step we vectorise the data using TF-IDF (Term Frequency - Inverse Document Frequency). There are other options, but we will be employing the use of TF-IDF. The convenient thing about using TF-IDF is that it can work on un-tokenized data.

  2. Split into training and testing data

    3. The data has to be split into training and testing data so that we can build the model and test it and evaluate its metrics. It is convenient that the data is already split into training and testing.

    NOTE: we dropped any "nan" values in taskB and taskC as they were not required for building the model. For example if a tweet was NOT OFFENSIVE, this meant that it cannot be TARGETED at a GROUP, INDIVIDUAL or OTHER, therefore a NULL value was placed there, but when loading the data in, it was converted to "nan" by the pandas library.

  3. Build and Evaluate the Model

    4. Next step was to build multiple models using the "sklearn" library, we built multiple models and evaluated them based on accuracy, precision and the confusion matrix. The following models were created and evaluated:

    • Dummy Classifier
    • Naive Bayes
    • SVC
    • Logistic Regression
    • Random Forest
    • Decision Tree
    • Gradient Boosting
    • Voted Perceptron
    • Convolution Neural Network

    To set a benchmark for the models, we used the Dummy classifier. If the models accuracy surpasses the Dummy classifier, it can be considered a success, and if it doesn't, it is considered a fail.

    To optimise the models, we did use GridSearchCV to search for the best paramaters (this code has been commented and can be run by uncommenting it), We only only selected the parameters that made a significant difference in the accuracy results

  4. Save the best model and the tokeniser to file for re-use

    5. Based on the best accuracy score, we stored that model into file using the pickle library. We also stored the tokeniser, which allowed for the model to be loaded in and then predicted against new data.
